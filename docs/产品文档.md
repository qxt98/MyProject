## 1. 产品概述

### 1.1 产品名称与定位

- **产品名称**：药品信息智能化管理系统  
- **定位**：面向医院药剂科、采购、审核及医护人员的药品信息与库存管理、采购审批、处方审核及患者用药指导的 Web 管理平台，满足本科毕业设计要求的完整业务闭环。

### 1.2 目标用户

| 用户角色     | 使用场景简述 |
|--------------|--------------|
| 系统管理员   | 用户与权限管理、系统参数、操作日志查看 |
| 药剂科人员   | 药品信息维护、入库/出库/退库、库存与流水查询、库存预警 |
| 采购人员     | 提交采购申请、查看申请状态与审批结果 |
| 审核人员     | 审批采购申请、审核处方（通过/驳回）、填写审批意见 |
| 医生         | 录入处方（含患者病情）、添加处方药品（需查看药品列表）、查看处方状态 |
| 护士         | 出库发药、处方查询（按实际权限） |
| 患者/访客    | 使用用药指导进行症状或用药咨询（仅供参考） |

### 1.3 核心价值

- 药品基础信息数字化、规范化，支持按名称与类别查询与维护。  
- 库存按药品与批次管理，入库/出库有流水可追溯，支持低库存与近效期预警。  
- 采购与处方流程线上化，状态可查、审批可留痕。  
- 患者用药指导支持规则与 AI（如 DeepSeek）双模式，采用**流式回答**边生成边展示，减少等待与超时；未登录用户按 IP 限流（如 3 次/小时），并明确“仅供参考、请遵医嘱”的免责说明。

## 2. 功能列表与说明

### 2.1 药品信息管理（需求 3.1）

| 功能点     | 说明 |
|------------|------|
| 列表查询   | 按药品名称（模糊）、类别（精确）筛选，分页展示；可选包含已停用药品。 |
| 新增药品   | 录入名称、类别、剂型规格、适应症、生产厂家、供应商、采购价、销售价、图片 URL 等。 |
| 编辑药品   | 修改已有药品信息，保证数据一致性。 |
| 删除药品   | 物理删除（若被库存/处方引用需在业务层做校验或改为逻辑删除）。 |
| 停用       | 通过“是否停用”字段实现逻辑停用，列表默认不展示停用药品。 |

### 2.2 库存管理（需求 3.2）

| 功能点     | 说明 |
|------------|------|
| 入库登记   | 选择药品、填写批次号、数量、生产日期、有效期、备注；系统自动增加或创建对应库存记录并写入入库流水。 |
| 出库登记   | 选择药品与批次、数量；系统扣减库存并写入出库流水，库存不足时提示。 |
| 入库流水   | 按药品、时间分页查询入库记录。 |
| 出库流水   | 按药品、时间分页查询出库记录。 |
| 库存查询   | 按药品查看各批次库存（数量、生产日期、有效期）。 |
| 低库存预警 | 库存数量低于设定阈值时在预警列表中展示。 |
| 近效期预警 | 效期在设定天数内的批次在预警列表中展示。 |

### 2.3 采购审批（需求 3.3）

| 功能点     | 说明 |
|------------|------|
| 提交申请   | 填写药品、数量、供应商、预算金额、采购理由等，提交后状态为“待审批”。 |
| 申请列表   | 按状态（待审批/已通过/已驳回）、时间分页查询。 |
| 审批通过   | 审核人员将状态改为“已通过”，记录审批人与时间。 |
| 驳回       | 审核人员将状态改为“已驳回”，填写驳回原因。 |

### 2.4 处方审核（需求 3.4）

| 功能点     | 说明 |
|------------|------|
| 新建处方   | 录入患者姓名、开方医生姓名、**患者病情信息（选填，主诉、诊断等，供审核参考）**，状态为“草稿”。 |
| 处方明细   | 为处方添加药品明细：从药品列表中选择药品、填写剂量、给药途径、用药时长；医生角色可查看药品列表以完成添加。 |
| 提交处方   | 草稿提交后状态为“已提交”，进入待审核。 |
| 审核通过   | 审核人员填写审核意见，状态改为“已通过”。 |
| 审核驳回   | 审核人员填写意见，状态改为“已驳回”。 |
| 处方列表   | 按患者、医生、状态、时间分页查询；列表展示患者病情摘要（前若干字），详情与明细弹窗中完整展示患者病情信息。 |

### 2.5 患者用药指导（需求 3.5）

| 功能点     | 说明 |
|------------|------|
| 提问       | 用户输入症状、用药或问题描述，长度限制（如 500 字）防止滥用。 |
| 智能回复   | 优先调用 AI（OpenAI 兼容接口，可配置 DeepSeek 等），未配置或失败时使用规则/关键词兜底；回复为**流式输出**，边生成边展示，避免长时间等待与超时。 |
| 免责声明   | 界面与回复中明确“仅供参考，不能替代诊疗，请遵医嘱”。 |
| 未登录限流 | 未登录用户按 IP 在时间窗口内（默认 1 小时）限次提问（默认 3 次），超限提示并引导登录或稍后再试；已登录用户不限。 |
| 剩余次数   | 未登录时展示本小时内剩余提问次数，超限时展示明确错误信息与登录入口。 |

### 2.6 系统管理（需求 3.6）

| 功能点     | 说明 |
|------------|------|
| 登录       | 用户名+密码校验，密码 BCrypt 加密存储。 |
| 用户列表   | 查看所有用户（不含密码）。 |
| 新增用户   | 录入用户名、密码、姓名、角色、是否启用。 |
| 编辑用户   | 修改姓名、角色、启用状态，可选修改密码。 |
| 角色       | 通过用户表的 role 字段实现简单 RBAC，后续可扩展角色-权限表与菜单控制。 |

## 3. 界面与交互说明

### 3.1 整体布局

- **顶部**：系统标题“药品信息智能化管理系统”及主导航（首页、药品信息、库存管理、采购审批、处方审核、用药指导、系统管理）。  
- **主区域**：根据路由渲染对应功能页面，列表页含查询条件、按钮（新增/查询等）及表格或卡片。

### 3.2 主要页面

- **首页**：欢迎语与各模块入口说明。  
- **药品信息**：查询框（名称、类别）、查询/新增按钮、药品列表表格（ID、名称、类别、规格、采购价、销售价、操作：编辑/删除）。  
- **库存管理**：入库/出库表单或入口、入库/出库流水分页列表、按药品查库存、低库存与近效期预警列表。  
- **采购审批**：采购申请列表（状态筛选）、新增申请、详情与审批通过/驳回操作。  
- **处方审核**：处方列表（患者/医生/状态筛选，含患者病情摘要列）、新建处方（含患者病情信息录入）、处方明细维护（含患者病情展示与药品选择）、详情与明细中展示患者病情、提交、审核通过/驳回；医生登录后可查看药品列表以便添加处方药品。  
- **用药指导**：输入框（含字数提示 0/500）、提交按钮；回复区采用**流式展示**（内容逐段追加、生成中显示光标动画）；免责声明固定展示于回复下方；未登录时展示剩余提问次数与限流说明，超限时展示错误卡片与登录引导。  
- **系统管理**：用户列表、新增/编辑用户（含角色选择）。  

### 3.3 交互原则

- 关键操作（删除、审批驳回）有二次确认或明确提示。  
- 接口失败时通过前端统一拦截提示错误信息。  
- 列表支持分页，避免单页数据量过大。

## 4. 非功能需求（产品视角）

- **性能**：列表与查询在常规数据量下响应时间可接受（设计文档中有分页与索引）。  
- **安全**：登录校验、密码加密、按角色控制功能（RBAC）的扩展预留。  
- **兼容**：支持主流浏览器（Chrome、Edge、Firefox 等）近期版本。  
- **可用性**：界面简洁、操作有提示，符合毕业设计答辩与演示要求。
