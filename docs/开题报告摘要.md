# 开题报告摘要

## 设计题目

基于 Spring Boot 的药品信息智能化管理系统的设计与实现

## 类型

软件开发类

## 一、选题背景与意义（简述）

- **国内**：医疗信息化推进，药品管理从手工向数字化转变；现有系统在智能化与扩展性上仍有提升空间。
- **国外**：微服务与分布式架构、数据分析与智能预测较成熟；因医疗体系与成本差异，通用性与成本优化是趋势。
- **复杂工程问题**：综合 Web、数据库与医疗业务；数据随机性与多约束（法规、性能、安全、成本）；多模块集成与工程判断、设计取舍及创新需求。

## 二、设计思路与主要内容

- **思路**：高效、安全、易用；以业务需求为导向，模块化设计；UML 建模；Vue + Spring Boot + MySQL；迭代开发 + CI/CD；多级测试。
- **主要模块**：药品信息管理、库存管理、采购审批、处方审核、患者用药指导、系统管理；性能优化与安全设计（索引、缓存、RBAC、加密）。

## 三、所用方法（技术选型）

- 前端：Vue.js（组件化、双向绑定、单页应用）。
- 后端：Spring Boot（业务逻辑、接口、内嵌 Tomcat）。
- 数据库：MySQL（事务、一致性、查询性能）。

## 四、可行性分析

- 技术可行：技术栈成熟，开发者具备相关基础。
- 操作可行：界面简洁、有操作指引。
- 经济可行：开源技术，成本可控。
- 法律可行：符合网络安全与个人信息保护相关法规。
- 社会可行：支撑智慧医疗与医院信息化需求。
- 环境可行：减少纸质文书，利于可持续发展。
