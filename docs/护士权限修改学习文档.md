# 护士权限修改学习文档

## 1. 需求变更说明

**原始需求**：护士账号具有库存管理权限和处方审核模块的查询权限（仅读）。

**变更后需求**：护士账号不能有库存管理、处方审核模块的权限。

**变更原因**：根据业务需求调整，护士角色不应参与库存管理和处方审核相关操作。

## 2. 修改的文件列表

本次修改涉及以下文件：

### 后端文件
1. `backend/src/main/java/com/smartpharma/config/SecurityConfig.java` - Spring Security权限配置

### 前端文件
1. `frontend/src/stores/auth.js` - 前端路由权限配置

### 文档文件
1. `docs/角色与功能对应说明.md` - 角色与权限说明文档

## 3. 各文件修改详情

### 3.1 SecurityConfig.java 修改

**文件路径**: `backend/src/main/java/com/smartpharma/config/SecurityConfig.java`

**修改内容**：
- 第59行：将处方审核GET请求的权限从`hasAnyRole("ADMIN", "REVIEWER", "DOCTOR", "NURSE")`修改为`hasAnyRole("ADMIN", "REVIEWER", "DOCTOR")`
- 移除了护士对处方审核模块的查询权限

**修改前**：
```java
.requestMatchers(HttpMethod.GET, "/prescription", "/prescription/**", "/api/prescription", "/api/prescription/**").hasAnyRole("ADMIN", "REVIEWER", "DOCTOR", "NURSE")
```

**修改后**：
```java
.requestMatchers(HttpMethod.GET, "/prescription", "/prescription/**", "/api/prescription", "/api/prescription/**").hasAnyRole("ADMIN", "REVIEWER", "DOCTOR")
```

**说明**：库存管理权限原本就不包含NURSE角色，所以不需要修改。

### 3.2 auth.js 修改

**文件路径**: `frontend/src/stores/auth.js`

**修改内容**：
- 第17行：将`/prescription`路由的允许角色从`['ADMIN', 'REVIEWER', 'DOCTOR', 'NURSE']`修改为`['ADMIN', 'REVIEWER', 'DOCTOR']`
- 移除了护士对处方审核页面的前端路由访问权限

**修改前**：
```javascript
'/prescription': ['ADMIN', 'REVIEWER', 'DOCTOR', 'NURSE'],
```

**修改后**：
```javascript
'/prescription': ['ADMIN', 'REVIEWER', 'DOCTOR'],
```

**说明**：`/stock`路由原本就不包含NURSE角色，所以不需要修改。

## 4. 修改思考逻辑

### 4.1 权限控制原则

本次修改遵循以下原则：
1. **最小权限原则**：只授予用户完成工作所必需的最小权限
2. **前后端双重验证**：后端接口权限控制 + 前端路由权限控制
3. **文档同步更新**：代码修改后同步更新相关文档

### 4.2 修改步骤分析

1. **分析当前权限配置**：
   - 查看SecurityConfig.java了解后端权限配置
   - 查看auth.js了解前端路由权限配置
   - 查看角色与功能对应说明.md了解文档状态

2. **识别需要修改的点**：
   - 后端：处方审核GET权限包含NURSE，需要移除
   - 前端：/prescription路由包含NURSE，需要移除
   - 文档：多处提到护士权限，需要同步更新

3. **实施修改**：
   - 先修改后端配置，确保API层权限正确
   - 再修改前端配置，确保路由访问控制正确
   - 最后更新文档，保持文档与代码一致

### 4.3 注意事项

1. **库存管理权限**：经检查，库存管理权限原本就不包含NURSE角色，所以不需要修改
2. **处方审核权限**：需要移除NURSE对GET请求的权限，POST等操作原本就不包含NURSE
3. **前端组件逻辑**：Prescription.vue中有`isPrescriptionReadOnly`计算属性，但护士已被移除路由权限，该属性逻辑可保持不变
4. **单元测试**：需要更新相关单元测试以反映权限变更
