## 1. 系统架构概述

### 1.1 总体架构

本系统采用**前后端分离**的 B/S 架构：

- **表现层**：Vue.js 单页应用（SPA），负责用户交互与页面展示，通过 HTTP 调用后端 REST API。
- **业务层**：Spring Boot 应用，提供 REST 接口，实现业务逻辑、事务控制与权限控制。
- **数据层**：MySQL 关系型数据库，持久化药品、库存、采购、处方、用户等数据。

```
┌─────────────┐     HTTP/JSON      ┌─────────────────┐     JDBC/JPA      ┌──────────┐
│  浏览器      │ ◄────────────────► │  Spring Boot     │ ◄────────────────► │  MySQL   │
│  Vue.js     │     /api/*         │  (Controller →   │                    │          │
│             │                    │   Service → Repo)│                    │          │
└─────────────┘                    └─────────────────┘                    └──────────┘
```

### 1.2 技术选型

| 层次   | 技术           | 版本/说明 |
|--------|----------------|-----------|
| 前端   | Vue.js         | 3.x，组合式 API，Vite 构建 |
| 路由   | Vue Router     | 4.x |
| 状态   | Pinia          | 2.x，存储登录态与用户信息、前端权限判断 |
| 请求   | Axios + fetch  | 常规接口用 Axios 封装；用药指导流式用 fetch 消费 SSE |
| 后端   | Spring Boot    | 3.2，JDK 17 |
| ORM    | Spring Data JPA | 与 Hibernate |
| 安全   | Spring Security + JWT | 密码 BCrypt，JWT 无状态认证，按角色控制接口与菜单 |
| API 文档 | springdoc-openapi | OpenAPI 3 + Swagger UI，便于查看与调试接口 |
| 数据库 | MySQL          | 8.0，utf8mb4 |

### 1.3 部署架构

- **开发环境**：前端 `npm run dev`（Vite 代理 /api 到后端），后端 `mvn spring-boot:run`，本地 MySQL 或 Docker MySQL。
- **Docker 部署**：docker-compose 编排 mysql、backend、frontend（Nginx）三服务；前端通过 Nginx 反向代理 /api 到 backend。

## 2. 功能模块设计

### 2.1 模块划分

| 模块           | 对应需求   | 后端入口                     | 前端路由/视图      |
|----------------|------------|------------------------------|--------------------|
| 药品信息管理   | 3.1        | /api/drugs                   | /drugs, DrugList   |
| 库存管理       | 3.2        | /api/stock                   | /stock, Stock      |
| 采购审批       | 3.3        | /api/purchase                | /purchase, Purchase|
| 处方审核       | 3.4        | /api/prescription            | /prescription      |
| 患者用药指导   | 3.5        | /api/guide                   | /guide, Guide      |
| 系统管理       | 3.6        | /api/auth, /api/user        | /system, System   |

### 2.2 分层结构（后端）

- **Controller**：接收 HTTP 请求，参数校验，调用 Service，返回 `Result<T>` 统一结构。
- **Service**：业务逻辑、事务边界（@Transactional）、调用 Repository。
- **Repository**：JpaRepository 及自定义查询，与数据库交互。
- **Entity**：JPA 实体，与表结构一一对应。
- **Config**：WebConfig（CORS）、SecurityConfig（密码加密、放行策略）、GuideAiProperties / GuideLimitProperties（用药指导 AI 与限流配置）、RestTemplateConfig（RestTemplate、配置属性注册）。
- **common**：Result 统一响应封装。
- **患者用药指导**：GuideController 调用 GuideAiService（OpenAI 兼容接口，可接 DeepSeek 等；支持一次性 ask 与流式 streamAsk）、GuideAskLimitService（匿名按 IP 限流）；规则匹配兜底；问题长度由 GuideLimitProperties 配置（如 500 字）。流式回答通过 SseEmitter 推送 content/disclaimer/remainingAsks 事件，前端边收边展示以提升体验、避免长时间等待与超时。

## 3. 数据库设计

### 3.1 主要表结构

| 表名              | 说明           | 关键字段 |
|-------------------|----------------|----------|
| drug              | 药品信息       | id, name, category, dosage_form, purchase_price, sale_price, disabled |
| stock             | 库存（药品+批次）| id, drug_id, batch_no, quantity, production_date, expiry_date |
| stock_in          | 入库记录       | id, drug_id, batch_no, quantity, created_at |
| stock_out         | 出库记录       | id, drug_id, batch_no, quantity, created_at |
| purchase_request  | 采购申请       | id, drug_id, quantity, supplier, status, approved_at, approved_by |
| prescription      | 处方主表       | id, patient_name, doctor_name, patient_condition（患者病情信息）, status, review_remark |
| prescription_item | 处方明细       | id, prescription_id, drug_id, dosage, route, duration |
| sys_user          | 系统用户       | id, username, password, real_name, role, enabled |
| operation_log     | 操作日志       | id, user_id, module, action, created_at（可选实现） |

### 3.2 关键约束与索引

- **stock**：(drug_id, batch_no) 唯一，便于入库合并、出库扣减。
- **drug**：name、category 建索引，支持列表条件查询。
- **purchase_request、prescription**：status、created_at 建索引，支持按状态与时间查询。

## 4. 接口设计

### 4.1 统一响应格式

- **常规接口**：JSON 体 `{ "code": 200, "message": "success", "data": { ... } }`；code=200 表示成功，非 200 表示失败，message 为错误说明。
- **用药指导流式**：`POST /api/guide/ask/stream` 返回 `Content-Type: text/event-stream`，按 SSE 规范推送事件（event: content/disclaimer/remainingAsks/error），无统一 JSON 包裹。

### 4.2 主要接口列表

| 模块     | 方法 | 路径示例              | 说明 |
|----------|------|------------------------|------|
| 药品     | GET  | /api/drugs             | 分页列表，参数 name, category, page, size；ADMIN/PHARMACIST/PURCHASER/REVIEWER/**DOCTOR** 可读（医生用于处方选药） |
| 药品     | GET  | /api/drugs/{id}        | 详情 |
| 药品     | POST | /api/drugs             | 新增 |
| 药品     | PUT  | /api/drugs/{id}        | 修改 |
| 药品     | DELETE | /api/drugs/{id}      | 删除 |
| 库存     | POST | /api/stock/in          | 入库 |
| 库存     | POST | /api/stock/out         | 出库 |
| 库存     | GET  | /api/stock/list        | 按药品查库存（drugId） |
| 库存     | GET  | /api/stock/in/list     | 入库流水分页 |
| 库存     | GET  | /api/stock/warn/low    | 低库存预警 |
| 采购     | GET  | /api/purchase          | 分页列表，参数 status |
| 采购     | POST | /api/purchase          | 提交申请 |
| 采购     | POST | /api/purchase/{id}/approve | 审批通过 |
| 采购     | POST | /api/purchase/{id}/reject  | 驳回 |
| 处方     | GET  | /api/prescription      | 分页列表，返回含 patient_condition |
| 处方     | GET  | /api/prescription/{id} | 处方详情，含 patient_condition |
| 处方     | POST | /api/prescription      | 新建处方，body 可含 patientName, doctorName, **patientCondition** |
| 处方     | POST | /api/prescription/{id}/submit | 提交 |
| 处方     | POST | /api/prescription/{id}/approve | 审核通过 |
| 用药指导 | POST | /api/guide/ask          | 提问（一次性返回），body: { question }；**匿名放行**，未登录按 IP 限流（默认 3 次/小时），响应含 answer、disclaimer，匿名时含 remainingAsks；超限返回 code=429 |
| 用药指导 | POST | /api/guide/ask/stream   | 提问（**流式**），body: { question }，响应 Content-Type: text/event-stream；推送事件 content（内容片段）、disclaimer（免责声明）、remainingAsks（匿名剩余次数）、error（错误）；前端默认使用此接口实现边生成边展示 |
| 认证     | POST | /api/auth/login        | 登录，body: { username, password }，返回 { token, user } |
| 认证     | GET  | /api/auth/me           | 获取当前登录用户（需携带 Authorization: Bearer &lt;token&gt;） |
| 用户     | GET  | /api/user              | 用户列表（仅 ADMIN） |
| 用户     | POST | /api/user              | 新增用户 |
| 用户     | PUT  | /api/user/{id}         | 修改用户 |
| 用户     | DELETE | /api/user/{id}       | 删除用户（仅 ADMIN） |
| 操作日志 | GET  | /api/operation-log     | 分页查询，参数 module、userId、page、size（仅 ADMIN） |

### 4.3 API 文档（Swagger / OpenAPI 3）

后端集成 **springdoc-openapi**，提供可浏览与调试的接口文档：

| 内容 | 路径（相对 context-path /api） | 说明 |
|------|--------------------------------|------|
| Swagger UI | /api/swagger-ui.html | 可视化接口列表与“试运行”调试 |
| OpenAPI 3 规范（JSON） | /api/v3/api-docs | 机器可读的接口定义，可供代码生成或导入 Postman 等 |

- **实现**：依赖 `springdoc-openapi-starter-webmvc-ui`，配置类 `config/OpenApiConfig.java` 设置文档标题与描述，`application.yml` 中 `springdoc` 配置路径。
- **访问**：后端启动后，浏览器访问 http://localhost:8080/api/swagger-ui.html；Docker 部署时也可通过前端代理 http://localhost:3000/api/swagger-ui.html。
- **安全**：开发阶段与接口一并放行；生产环境建议通过配置或安全策略关闭或限制 Swagger 访问。

## 5. 安全与性能设计

### 5.1 安全与权限（用户角色与登录）

- **认证**：登录接口 `POST /api/auth/login` 校验用户名与 BCrypt 密码，成功后返回 **JWT token** 与用户信息（含角色）。前端将 token 存入本地，请求时在请求头携带 `Authorization: Bearer <token>`；后端通过 `JwtAuthFilter` 解析 token 并设置 Spring Security 上下文。
- **角色（RBAC）**：User 表 `role` 字段取值见下表，对应中国医院常见岗位；不同角色可访问的接口与前端菜单不同。

| 角色 code   | 说明   | 可访问模块（后端接口 / 前端菜单） |
|-------------|--------|----------------------------------|
| ADMIN       | 管理员 | 全部：系统管理、药品、库存、采购、处方、用药指导 |
| PHARMACIST  | 药师   | 药品信息、库存管理、用药指导 |
| PURCHASER   | 采购员 | 库存管理、采购审批、用药指导 |
| REVIEWER    | 审核员 | 采购审批、处方审核、用药指导 |
| DOCTOR      | 医生   | 处方审核（开立/查看）、**药品列表只读（用于处方选药）**、用药指导 |
| NURSE       | 护士   | 库存管理、处方查询（只读）、用药指导 |

- **接口权限**：除 `/auth/login`、Swagger 文档、根路径放行外，其余接口需已认证；`/user/**` 仅 ADMIN；**GET /drugs、/drugs/** 为 ADMIN、PHARMACIST、PURCHASER、REVIEWER、**DOCTOR**（医生仅读，用于处方选药）；其余 `/drugs/**` 写操作为 ADMIN、PHARMACIST；`/stock/**` 为 ADMIN、PHARMACIST、PURCHASER、NURSE；`/purchase/**` 为 ADMIN、PURCHASER、REVIEWER；`/prescription/**` 为 ADMIN、REVIEWER、DOCTOR（护士仅 GET）。**患者用药指导**：`POST /guide/ask`、`POST /guide/ask/stream` 及对应 `/api` 前缀设为 **permitAll()**，供未登录患者/访客匿名咨询；其余 `/guide/**` 需已登录。匿名访问时由 `GuideAskLimitService` 按 IP 滑动窗口限流（默认 3 次/小时），超限返回 429 或 SSE 的 error 事件；已登录用户不限。详见后端 `SecurityConfig`。
- **前端权限**：路由守卫未登录跳转登录页；按 `ROUTE_ROLES` 控制各 path 可访问角色，无权限时跳转首页；导航栏仅展示当前角色可访问的菜单。
- **数据**：密码不明文存储；敏感接口与操作可记录操作日志。

### 5.2 性能

- 列表接口均支持分页，避免一次加载过多数据。
- 数据库表建索引（药品名称、类别、库存药品+批次、采购/处方状态与时间）。
- 可按需对热点查询（如药品列表）增加缓存（如 Redis）。
