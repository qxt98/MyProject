# 药品信息智能化管理系统 — 设计说明

## 1. 设计原则

- **高效**：减少重复操作，支持批量与快捷操作，关键查询与列表做性能优化。
- **安全**：RBAC 权限、敏感数据加密、操作日志与审计。
- **易用**：界面简洁、流程清晰、关键步骤有提示与校验。

## 2. 系统架构

- **前端**：Vue.js 单页应用（SPA），组件化开发，通过 Ajax 与后端 REST API 通信。
- **后端**：Spring Boot 分层架构：
  - Controller：接收请求、参数校验、返回统一响应。
  - Service：业务逻辑、流程编排、事务控制。
  - Repository/DAO：数据访问，与 MySQL 交互。
  - API 文档：集成 springdoc-openapi（Swagger UI），提供接口查看与调试（见开发与部署说明）。
- **数据库**：MySQL，事务保证一致性，合理使用索引与缓存提升查询性能。

## 3. 功能模块设计概要

### 3.1 药品信息管理

- 实体：药品名称、类别、剂型规格、适应症、生产厂家、供应商、购销价格、图片等。
- 功能：列表、条件查询、新增、编辑、删除（逻辑删除建议）、导入导出（可选）。

### 3.2 库存管理

- 实体：库存表、入库单、出库单、退库单，关联药品、批次、生产日期、有效期、数量。
- 功能：入库/出库/退库登记、库存流水查询、库存预警（低库存、近效期）。

### 3.3 采购审批

- 实体：采购申请（药品、数量、供应商、预算、理由、状态）、审批记录。
- 功能：提交申请、审批/驳回、状态流转、历史查询。

### 3.4 处方审核

- 实体：处方单、处方明细（药品、剂量、途径、频次、时长）、审核结果、风险项。
- 功能：处方录入/导入、与药品库比对、剂量与配伍校验、风险提示、审核通过/不通过。

### 3.5 患者用药指导

- 功能：用户输入症状/病史/当前用药，系统结合药品库与用药规范做语义分析与回复，给出用药建议（可对接简单规则引擎或后续扩展 AI）。

### 3.6 系统管理

- 用户管理：账号、姓名、角色、状态。
- 角色与权限：RBAC，角色-菜单/接口权限绑定。
- 操作日志：关键操作记录（谁、何时、做了什么）。
- 系统参数：如库存预警阈值、审批超时时间等可配置项。

## 4. 性能与安全设计

- **性能**：关键表建索引、避免 N+1 查询、热点数据可加缓存（如 Redis）；列表分页、条件精简。
- **安全**：登录态与 Token、接口级权限校验、密码加密存储、敏感字段加密与脱敏；遵循《网络安全法》《个人信息保护法》等要求。

## 5. 开发与迭代

- 采用迭代开发，先核心模块（药品、库存、用户权限），再采购审批、处方审核、用药指导。
- 结合 CI/CD 做自动构建与部署；通过单元测试、集成测试、系统测试及 UAT 保证质量。
