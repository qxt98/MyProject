# 开发与部署说明

## 一、使用 Docker Desktop 部署（推荐）

本系统支持在 **Docker Desktop** 中一键运行数据库与前后端服务，无需在本机安装 JDK、Node、MySQL。

> **详细说明**：Docker 相关文件说明、服务与端口、环境变量、数据持久化、常见问题等见 **[Docker 使用说明](Docker使用说明.md)**。

### 1.1 环境要求

- 已安装 [Docker Desktop](https://www.docker.com/products/docker-desktop/)（Windows / Mac 均可）
- 确保 Docker 引擎已启动，且内存等资源充足（建议至少 4GB）

### 1.2 启动项目

在项目根目录（与 `docker-compose.yml` 同级）执行：

```bash
docker compose up -d
```

首次运行会拉取 MySQL 镜像并构建后端、前端镜像，可能需要几分钟。完成后：

- **前端（浏览器访问）**：http://localhost:3000（工作人员需登录；**患者用药指导**无需登录，可直接访问 http://localhost:3000/guide 或从登录页点击「患者用药指导（无需登录）」）
- **后端 API**：http://localhost:8080/api（前端页面的 `/api` 请求会由 Nginx 代理到后端）
- **Swagger 接口文档**：http://localhost:8080/api/swagger-ui.html（或 http://localhost:3000/api/swagger-ui.html 经 Nginx 代理）

> 若本机 80 端口未被占用，可在 `docker-compose.yml` 中将 frontend 的 `ports` 改为 `"80:80"`，即可通过 http://localhost 访问。

### 1.3 可选：自定义数据库密码

复制环境变量示例并修改后，再执行 `docker compose up -d`：

```bash
copy .env.example .env
# 编辑 .env，修改 MYSQL_ROOT_PASSWORD、MYSQL_PASSWORD 等；可选配置 GUIDE_AI_API_KEY 启用患者用药指导 AI（见第六节）
docker compose up -d
```

### 1.4 常用命令

| 操作       | 命令 |
|------------|------|
| 查看运行状态 | `docker compose ps` |
| 查看日志     | `docker compose logs -f`（或加服务名如 `backend`） |
| 停止所有服务 | `docker compose down` |
| 重新构建并启动 | `docker compose up -d --build` |

### 1.5 数据持久化

MySQL 数据已通过 Docker 卷 `mysql_data` 持久化，执行 `docker compose down` 不会删除数据库数据。若需清空数据并重新初始化（包括重新执行建表脚本），可执行：

```bash
docker compose down -v
docker compose up -d
```

（`-v` 会删除 compose 中定义的 volume，其中包含 MySQL 数据；再次 `up` 时会重新创建库并执行 `schema.sql`。）

## 二、本地开发环境（非 Docker）

### 2.1 环境准备

- **JDK 17+**
- **Node.js 18+**（推荐 LTS）
- **MySQL 8.0+**
- **Maven 3.8+**

### 2.2 数据库

1. 创建数据库：

```sql
CREATE DATABASE smart_pharma DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

2. 执行建表脚本（需先创建库后再执行，脚本内含 `USE smart_pharma;`）：

```bash
mysql -u root -p smart_pharma < database/schema.sql
```

3. 在 `backend/src/main/resources/application.yml`（或 `application-dev.yml`）中配置数据库连接：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/smart_pharma?...
    username: root
    password: 你的密码
```

### 2.3 后端（Spring Boot）

```bash
cd backend
mvn clean spring-boot:run
```

- 开发时可使用 profile：`mvn spring-boot:run -Dspring-boot.run.profiles=dev`
- 默认端口：8080，上下文路径：`/api`，故接口基础地址为 `http://localhost:8080/api`。
- **Swagger 文档**：启动后访问 http://localhost:8080/api/swagger-ui.html 可查看与调试接口；OpenAPI 规范地址为 http://localhost:8080/api/v3/api-docs。

### 2.4 前端（Vue + Vite）

```bash
cd frontend
npm install
npm run dev
```

- 开发服务器默认端口：5173。
- Vite 已配置代理：`/api` 转发到 `http://localhost:8080`，前端请求 `/api/xxx` 即可访问后端。

## 三、生产构建（非 Docker）

- **后端**：`cd backend && mvn clean package`，得到 `target/smart-pharma-backend-1.0.0-SNAPSHOT.jar`，使用 `java -jar ...` 运行。
- **前端**：`cd frontend && npm run build`，产物在 `dist/`，可部署到 Nginx 等静态服务器；若与后端同域，需在 Nginx 中配置 `/api` 反向代理到 Spring Boot。

## 四、测试

- 后端单元测试：`cd backend && mvn test`
- 前端可后续接入 Vitest 等做单元/组件测试；系统测试与 UAT 按开题报告要求执行。

## 五、API 文档（Swagger）

后端集成 **springdoc-openapi**（OpenAPI 3 + Swagger UI），用于查看与调试 REST 接口：

| 内容       | 地址 |
|------------|------|
| Swagger UI | http://localhost:8080/api/swagger-ui.html |
| OpenAPI JSON | http://localhost:8080/api/v3/api-docs |

- 本地开发时直接访问上述地址；Docker 部署时也可通过前端代理访问：http://localhost:3000/api/swagger-ui.html。
- 配置位于 `backend` 模块：依赖见 `pom.xml`（springdoc-openapi-starter-webmvc-ui），文档信息见 `config/OpenApiConfig.java`，路径在 `application.yml` 的 `springdoc` 下。
- 生产环境建议关闭或限制 Swagger 访问（如通过配置或安全策略）。

## 六、患者用药指导 AI 配置（可选）

「患者用药指导」支持接入 **OpenAI 兼容** 的对话接口，未配置或调用失败时自动使用内置规则兜底。

### 6.1 配置项（`application.yml` 或环境变量）

| 配置项 | 说明 | 默认 |
|--------|------|------|
| `guide.ai.enabled` | 是否启用 AI | `true` |
| `guide.ai.api-key` | API Key（**勿提交到版本库**，建议用环境变量） | 空 |
| `guide.ai.base-url` | 接口地址 | `https://api.deepseek.com` |
| `guide.ai.model` | 模型名 | `deepseek-chat` |
| `guide.ai.max-tokens` | 最大生成长度 | `500` |

环境变量示例：`GUIDE_AI_API_KEY=sk-xxx`（在 `application.yml` 中已用 `api-key: ${GUIDE_AI_API_KEY:}` 引用）。**Docker 部署**时在项目根目录 `.env` 中配置该变量即可，`docker-compose.yml` 会将其传入后端容器。

### 6.2 推荐接入方式

- **DeepSeek**：国内可用、价格低、中文好。注册后获取 API Key，`base-url` 保持 `https://api.deepseek.com`，`model` 用 `deepseek-chat`。
- **智谱 GLM**：有免费额度，兼容 OpenAI 格式，将 `base-url` 改为智谱提供的地址并配置对应 `model`（如 `glm-4-flash`）和 API Key。
- **通义千问 / 文心等**：若提供 OpenAI 兼容的 chat 接口，只需改 `base-url`、`model` 和 `api-key`。
- **Ollama（本地）**：无需 API Key。安装 [Ollama](https://ollama.com/) 后运行模型（如 `ollama run qwen2.5:7b`），配置 `guide.ai.base-url: http://localhost:11434`、`guide.ai.model: qwen2.5:7b`，不配置或留空 `api-key` 即可。

不配置 API Key 且未使用本地 Ollama 时，系统不会发起外部请求，仅使用规则匹配回复。

### 6.3 未登录用户限流与安全

- **限流**：未登录用户按 **IP** 在 **滑动时间窗口**（默认 1 小时）内最多提问 **3 次**，超限返回 HTTP 200 + `code: 429` 及提示；已登录用户不限制。
- **配置**（`application.yml` 下 `guide.limit`）：`max-asks-per-window`（默认 3）、`window-hours`（默认 1）、`question-max-length`（默认 500 字符）。
- **安全**：问题内容做长度校验；响应中仅返回业务数据与免责声明，不暴露内部错误；限流采用内存存储（单机有效），重启或多实例各自计数。

## 七、注意事项

- 生产环境务必修改：数据库密码、JWT 密钥、关闭或限制 CORS、开启 HTTPS。
- 首次使用可在 `database/schema.sql` 中取消注释并修改管理员初始密码的 INSERT 语句，或通过系统管理界面创建用户。
- 患者用药指导的 AI Key 请勿写入版本库，建议通过环境变量或配置中心注入。
