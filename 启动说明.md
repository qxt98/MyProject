# Smart Pharma 项目启动说明

## 快速启动（推荐）

### 方法 1: 使用一键启动脚本

```bash
./start-with-ngrok.sh
```

这个脚本会：
1. 检查 Docker 和 ngrok 环境
2. 启动所有 Docker 服务（MySQL、后端、前端）
3. 启动 ngrok 隧道暴露前端服务

### 方法 2: 分步启动

#### 步骤 1: 启动 Docker 服务

```bash
docker compose up -d
```

如果遇到网络证书错误，请参考 `Docker网络问题解决方案.md`

#### 步骤 2: 验证服务运行

```bash
docker compose ps
```

应该看到三个服务：
- `smart-pharma-mysql` (端口 3306)
- `smart-pharma-backend` (端口 8080)
- `smart-pharma-frontend` (端口 3000)

#### 步骤 3: 启动 ngrok

```bash
./start-ngrok-only.sh
```

或者手动启动：

```bash
ngrok http 3000
```

## 访问地址

### 本地访问
- **前端**: http://localhost:3000
- **后端 API**: http://localhost:8080
- **MySQL**: localhost:3306

### 外网访问（通过 ngrok）

启动 ngrok 后，会显示类似以下的信息：

```
Forwarding   https://xxxx-xx-xx-xx-xx.ngrok-free.app -> http://localhost:3000
```

使用这个 `https://xxxx-xx-xx-xx-xx.ngrok-free.app` 地址即可在外网访问。

**注意**: 
- ngrok 免费版会显示警告页面，点击 "Visit Site" 即可继续
- 免费版地址每次启动都会变化
- 如需固定域名，需要升级 ngrok 账户

## 停止服务

### 停止所有服务（包括 ngrok）

```bash
# 停止 Docker 服务
docker compose down

# 停止 ngrok (如果在运行，按 Ctrl+C)
```

### 仅停止 Docker 服务

```bash
docker compose stop
```

### 仅停止 ngrok

在运行 ngrok 的终端按 `Ctrl+C`

## 查看日志

```bash
# 查看所有服务日志
docker compose logs -f

# 查看特定服务日志
docker compose logs -f backend
docker compose logs -f frontend
docker compose logs -f mysql
```

## 常见问题

### 1. Docker 网络证书错误

参考 `Docker网络问题解决方案.md` 文件

### 2. 端口被占用

如果 3000 或 8080 端口被占用：

```bash
# 检查端口占用
lsof -i :3000
lsof -i :8080

# 修改 docker-compose.yml 中的端口映射
# 例如将 "3000:80" 改为 "3001:80"
```

### 3. ngrok 认证

如果首次使用 ngrok，可能需要注册并配置 authtoken：

```bash
ngrok config add-authtoken YOUR_AUTHTOKEN
```

获取 authtoken: https://dashboard.ngrok.com/get-started/your-authtoken

### 4. 服务启动失败

检查服务状态和日志：

```bash
docker compose ps
docker compose logs backend
docker compose logs frontend
docker compose logs mysql
```

## 环境变量配置

如果需要配置环境变量，可以创建 `.env` 文件：

```env
MYSQL_ROOT_PASSWORD=your_password
MYSQL_USER=app
MYSQL_PASSWORD=app123
GUIDE_AI_API_KEY=your_api_key
```

## 数据库初始化

数据库会在首次启动时自动初始化（使用 `database/schema.sql`）

如果需要重新初始化：

```bash
docker compose down -v  # 删除数据卷
docker compose up -d     # 重新创建
```
