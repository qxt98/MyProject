# 解决 Docker 镜像源超时问题

## 当前问题
镜像源 `docker.domys.cc` 连接超时，导致无法拉取镜像。

## 解决方案

### 方案 1: 移除有问题的镜像源（推荐）

1. 打开 Docker Desktop
2. 进入 Settings > Docker Engine
3. 编辑配置，移除或注释掉 `docker.domys.cc`：

```json
{
  "registry-mirrors": [
    "https://docker.mirrors.ustc.edu.cn",
    "http://hub-mirror.c.163.com",
    "https://docker.m.daocloud.io"
  ]
}
```

4. 点击 "Apply & Restart"
5. 重新运行 `docker compose up -d`

### 方案 2: 使用官方 Docker Hub（如果网络允许）

如果在中国大陆外，可以临时移除所有镜像源，直接使用官方源：

```json
{
  "registry-mirrors": []
}
```

### 方案 3: 手动拉取镜像后构建

如果镜像源都不稳定，可以尝试：

```bash
# 尝试从不同源拉取基础镜像
docker pull mysql:8.0
docker pull maven:3.9-eclipse-temurin-17
docker pull node:20-alpine
docker pull nginx:alpine
docker pull eclipse-temurin:17-jre

# 如果拉取成功，再构建和启动
docker compose build
docker compose up -d
```

### 方案 4: 使用国内其他镜像源

可以尝试添加其他可用的镜像源：

```json
{
  "registry-mirrors": [
    "https://docker.mirrors.ustc.edu.cn/",
    "https://hub-mirror.c.163.com/",
    "https://mirror.baidubce.com"
  ]
}
```

## 快速修复脚本

运行以下命令可以尝试自动修复：

```bash
./fix-mirror-timeout.sh
```
