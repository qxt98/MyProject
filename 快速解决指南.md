# 快速解决 Docker 证书错误

## 问题现象
```
Error: tls: failed to verify certificate: x509: certificate is valid for *.facebook.com, ...
```

## 根本原因
Docker 配置了代理 `http.docker.internal:3128`，导致证书验证失败。

## 最快解决方案（3步）

### 步骤 1: 禁用 Docker 代理
1. 打开 **Docker Desktop**
2. 点击右上角 **Settings** (齿轮图标)
3. 左侧菜单选择 **Resources** > **Proxies**
4. 取消勾选 **"Manual proxy configuration"** 或清空所有代理字段
5. 点击 **"Apply & Restart"**
6. 等待 Docker 完全重启（约 30 秒）

### 步骤 2: 启动服务
```bash
cd /Users/haiyiguo/GitRepor/smart-pharma
docker compose up -d
```

等待 30-60 秒让服务完全启动。

### 步骤 3: 启动 ngrok
```bash
./start-ngrok-only.sh
```

或者：
```bash
ngrok http 3000
```

## 验证

检查服务状态：
```bash
./check-services.sh
```

或手动检查：
```bash
docker compose ps
```

应该看到三个服务都在运行：
- ✅ smart-pharma-mysql
- ✅ smart-pharma-backend
- ✅ smart-pharma-frontend

## 如果禁用代理后仍有问题

### 方案 A: 使用镜像加速器
1. Docker Desktop > Settings > Docker Engine
2. 添加以下配置（如果还没有）：
```json
{
  "registry-mirrors": [
    "https://docker.mirrors.ustc.edu.cn",
    "https://hub-mirror.c.163.com"
  ]
}
```
3. Apply & Restart

### 方案 B: 手动拉取镜像
```bash
# 逐个拉取基础镜像
docker pull mysql:8.0
docker pull maven:3.9-eclipse-temurin-17
docker pull node:20-alpine
docker pull nginx:alpine
docker pull eclipse-temurin:17-jre

# 然后构建和启动
docker compose build
docker compose up -d
```

## 获取外网访问地址

ngrok 启动后会显示：
```
Forwarding   https://xxxx-xx-xx-xx-xx.ngrok-free.app -> http://localhost:3000
```

复制这个 `https://xxxx-xx-xx-xx-xx.ngrok-free.app` 地址即可在外网访问。

**注意**: 
- 免费版首次访问会显示警告页面，点击 "Visit Site" 继续
- 地址每次启动都会变化
