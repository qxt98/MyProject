# Smart Pharma 项目本地运行指南

本文档记录在本地（不使用 Docker Desktop）运行 Smart Pharma 项目的完整过程，包括环境配置、镜像加速、服务启动和问题解决。

## 环境要求

- **操作系统**: Windows 11（其他系统类似）
- **Java**: 17（本项目使用 Spring Boot 3.2.0）
- **Maven**: 3.6.3+（用于构建后端）
- **Node.js**: 18+（用于运行前端）
- **MySQL**: 8.0+（数据库）

## 步骤概览

1. 检查并安装所需软件
2. 配置 Maven 和 npm 国内镜像
3. 启动 MySQL 并初始化数据库
4. 启动 Spring Boot 后端
5. 启动 Vue.js 前端
6. 验证服务并访问

## 详细步骤

### 1. 检查软件版本

确保以下命令能正确输出版本：

```bash
java -version
mvn --version
npm --version
mysql --version
```

### 2. 配置国内镜像

#### Maven 镜像（阿里云）

在用户主目录的 `.m2` 文件夹中创建 `settings.xml` 文件，内容如下：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<settings xmlns="http://maven.apache.org/SETTINGS/1.0.0"
          xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
          xsi:schemaLocation="http://maven.apache.org/SETTINGS/1.0.0 http://maven.apache.org/xsd/settings-1.0.0.xsd">
  <mirrors>
    <mirror>
      <id>aliyunmaven</id>
      <mirrorOf>*</mirrorOf>
      <name>阿里云公共仓库</name>
      <url>https://maven.aliyun.com/repository/public</url>
    </mirror>
  </mirrors>
</settings>
```

Windows PowerShell 中创建命令：

```powershell
if (!(Test-Path ~/.m2)) { New-Item -ItemType Directory -Path ~/.m2 -Force }; @'
<?xml version="1.0" encoding="UTF-8"?>
<settings xmlns="http://maven.apache.org/SETTINGS/1.0.0"
          xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
          xsi:schemaLocation="http://maven.apache.org/SETTINGS/1.0.0 http://maven.apache.org/xsd/settings-1.0.0.xsd">
  <mirrors>
    <mirror>
      <id>aliyunmaven</id>
      <mirrorOf>*</mirrorOf>
      <name>阿里云公共仓库</name>
      <url>https://maven.aliyun.com/repository/public</url>
    </mirror>
  </mirrors>
</settings>
'@ | Out-File -FilePath ~/.m2/settings.xml -Encoding utf8
```

#### npm 镜像（淘宝）

```bash
npm config set registry https://registry.npmmirror.com
```

### 3. 数据库准备

#### 启动 MySQL 服务

确保 MySQL 服务正在运行（Windows 服务中启动或使用 `net start mysql`）。

#### 创建数据库并初始化表结构

项目根目录下的 `database/schema.sql` 包含了完整的表结构定义。执行以下命令导入：

```bash
# 进入项目根目录
cd c:\project\smart-pharma-main

# 使用 PowerShell 导入 SQL 文件
Get-Content database/schema.sql | mysql -u root -p200498qiu smart_pharma
```

如果出现编码错误，可以直接在 MySQL 客户端中执行：

```sql
USE smart_pharma;
SOURCE database/schema.sql;
```

### 4. 启动后端服务

#### 修改 Hibernate 配置（可选）

如果数据库表结构与实体不匹配，需要将 `backend/src/main/resources/application.yml` 中的 `ddl-auto` 从 `validate` 改为 `update`：

```yaml
spring:
  jpa:
    hibernate:
      ddl-auto: update
```

#### 使用 Maven 启动 Spring Boot

在项目根目录执行：

```bash
cd backend
mvn spring-boot:run
```

或者使用 PowerShell 的 `pushd` 命令：

```powershell
pushd backend; mvn spring-boot:run
```

**注意**：如果使用 PowerShell，避免使用 `&&` 连接命令，因为 PowerShell 7 以下版本不支持。可以使用分号 `;` 分隔。

#### 验证后端启动

当看到以下日志时，表示后端启动成功：

```
Tomcat started on port 8080 (http) with context path '/api'
Started SmartPharmaApplication in X.XXX seconds
```

### 5. 启动前端服务

#### 检查依赖

前端 `node_modules` 目录应已存在（如果不存在，运行 `npm install`）。

#### 启动开发服务器

在项目根目录执行：

```bash
cd frontend
npm run dev
```

或使用 PowerShell：

```powershell
pushd frontend; npm run dev
```

前端默认运行在 `http://localhost:5173`，如果端口被占用会自动切换到其他端口（如 `5174`）。

### 6. 访问应用

- **前端页面**: http://localhost:5174/（根据实际端口调整）
- **后端 API**: http://localhost:8080/api
- **Swagger 文档**: http://localhost:8080/api/swagger-ui.html

#### 默认登录账号

- 用户名: `admin`
- 密码: `123456`

## 常见问题与解决

### 1. Maven 构建失败：依赖下载慢

已通过配置阿里云镜像解决。

### 2. Hibernate 启动失败：Schema-validation: missing column

错误信息示例：
```
Schema-validation: missing column [reject_reason] in table [purchase_request]
```

**解决方案**：将 `application.yml` 中的 `ddl-auto` 改为 `update`，让 Hibernate 自动更新表结构。

### 3. 端口占用

- 后端默认端口 `8080`，前端默认端口 `5173`。
- 如果端口被占用，可以修改 `application.yml` 中的 `server.port` 或前端 `vite.config.js` 中的配置。

### 4. MySQL 连接失败

检查 `application.yml` 中的数据库连接配置：
- `url`: `jdbc:mysql://localhost:3306/smart_pharma`
- `username`: `root`
- `password`: `200498qiu`

确保 MySQL 服务运行，且数据库 `smart_pharma` 存在。

### 5. PowerShell 中命令分隔符问题

PowerShell 中 `&&` 可能不被支持，使用分号 `;` 分隔多个命令，或使用 `pushd` 和 `popd`。

## 服务管理

### 启动服务（使用 server_runner 工具）

- 后端：`server_runner start --name backend --command "pushd backend; mvn spring-boot:run"`
- 前端：`server_runner start --name frontend --command "pushd frontend; npm run dev"`

### 查看日志

- 后端：`server_runner getLogs --name backend --lines 20`
- 前端：`server_runner getLogs --name frontend --lines 20`

### 停止服务

- 后端：`server_runner stop --name backend`
- 前端：`server_runner stop --name frontend`

## 总结

通过以上步骤，可以在不使用 Docker Desktop 的情况下成功在本地运行 Smart Pharma 项目。关键点包括配置国内镜像加速依赖下载、正确初始化数据库、解决 Hibernate 表结构验证问题。

此文档可作为后续团队成员的参考，快速搭建本地开发环境。
