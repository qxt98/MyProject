# 立即解决镜像源超时问题

## 问题
`docker.domys.cc` 镜像源超时，导致无法拉取镜像。

## 最快解决方案（2步）

### 步骤 1: 修改 Docker 镜像源配置

1. **打开 Docker Desktop**
2. **点击 Settings** (右上角齿轮图标)
3. **选择 Docker Engine** (左侧菜单)
4. **编辑配置**，移除有问题的 `docker.domys.cc`：

**推荐配置（使用稳定的镜像源）：**
```json
{
  "registry-mirrors": [
    "https://docker.mirrors.ustc.edu.cn",
    "http://hub-mirror.c.163.com",
    "https://docker.m.daocloud.io"
  ]
}
```

**或者如果网络允许，直接使用官方源：**
```json
{
  "registry-mirrors": []
}
```

5. **点击 "Apply & Restart"**
6. **等待 Docker 完全重启**（约 30 秒）

### 步骤 2: 重新启动服务

```bash
docker compose up -d
```

## 如果仍然超时

### 方法 A: 逐个手动拉取镜像

```bash
# 尝试拉取基础镜像（可能需要多次重试）
docker pull mysql:8.0
docker pull maven:3.9-eclipse-temurin-17  
docker pull node:20-alpine
docker pull nginx:alpine
docker pull eclipse-temurin:17-jre

# 如果拉取成功，再启动服务
docker compose build
docker compose up -d
```

### 方法 B: 使用其他镜像源

如果当前镜像源都不稳定，可以尝试添加：

```json
{
  "registry-mirrors": [
    "https://docker.mirrors.ustc.edu.cn/",
    "https://hub-mirror.c.163.com/",
    "https://mirror.baidubce.com"
  ]
}
```

### 方法 C: 检查网络连接

```bash
# 测试网络连接
ping -c 3 docker.mirrors.ustc.edu.cn
ping -c 3 registry-1.docker.io

# 如果 ping 不通，可能是网络问题
```

## 验证

配置修改后，检查是否生效：

```bash
docker info | grep -A 5 "Registry Mirrors"
```

然后尝试拉取一个测试镜像：

```bash
docker pull hello-world
```

如果成功，再运行：

```bash
docker compose up -d
```

## 启动 ngrok

服务启动成功后，启动 ngrok：

```bash
./start-ngrok-only.sh
```
